# components

## 1. 这个文件夹是干嘛的？（大白话）
这是一个存放组件代码的文件夹，主要用来存放 React 项目的各种组件。简单来说，就是项目中的各种界面元素，比如页面、卡片、弹窗等，都是由这里的组件代码实现的。

## 2. 里面存放哪些内容？
- **ChuQuChiPage.tsx**：出去吃页面组件
- **DiscussionPage.tsx**：讨论页面组件
- **HotkeysHelp.tsx**：热键帮助组件
- **LazyImage.tsx**：懒加载图片组件
- **PWAInstallPrompt.tsx**：PWA 安装提示组件
- **ProfilePage.tsx**：个人资料页面组件
- **RecipeCard.optimized.tsx**：优化后的菜谱卡片组件
- **RecipeCard.tsx**：菜谱卡片组件
- **RecipeGenerator.tsx**：菜谱生成器组件
- **Skeleton.tsx**：骨架屏组件
- **ToastContainer.tsx**：提示消息容器组件
- **VirtualList.tsx**：虚拟列表组件

## 3. 哪些文件绝对不能删？
- 所有的组件文件都不能删，否则对应的界面元素就无法显示
- RecipeCard.tsx / RecipeCard.optimized.tsx：菜谱卡片组件，删了就无法显示菜谱卡片
- RecipeGenerator.tsx：菜谱生成器组件，删了就无法生成菜谱
- ProfilePage.tsx：个人资料页面组件，删了就无法显示个人资料

## 4. 哪些是备份/旧版本/无用文件？
- RecipeCard.optimized.tsx：可能是 RecipeCard.tsx 的优化版本，两者可能只需要一个

## 5. 这个文件夹是否影响项目正常运行？
是的，非常影响。如果这个文件夹不存在或者组件文件有问题，项目的界面就无法正常显示。

## 6. 我可以在这里做什么？
- 修改现有的组件
- 添加新的组件
- 调整组件的布局和样式
- 优化组件的性能

## 7. 注意事项（非常重要）
- 不要删除核心组件文件，否则对应的界面元素无法显示
- 修改组件时，要确保功能正常
- 组件的命名要规范，便于理解和维护
- 可以使用组件化的方式来组织界面，提高代码的复用性
- 这个项目使用了 TypeScript，所以要注意类型定义
