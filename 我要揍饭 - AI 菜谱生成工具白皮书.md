## 执行摘要

### 一句话精准项目定位
"我要揍饭"是一款基于 AI 技术的食材优先智能菜谱工具，解决用户"家里有食材但不知道做什么"的日常高频痛点，核心优势是精准匹配现有食材、智能适配厨具、提供个性化菜谱推荐。

### 核心痛点总结
1. **食材反向匹配困难**：现有菜谱产品多以菜名为中心，用户难以快速找到使用现有食材的菜谱，常导致食材浪费
2. **厨具适配性差**：忽略用户实际拥有的厨具，推荐需要特定厨具的菜谱，导致用户无法实际操作
3. **调料信息不明确**：未清晰标注所需调料，用户在烹饪过程中发现缺少调料，影响烹饪体验

### 我们的解决方案
以食材为中心，结合用户拥有的厨具和调料，通过智能匹配和 AI 生成，为用户提供可立即执行的个性化菜谱。

### 当前开发进度
项目处于单人开发的 MVP 早期阶段，正在开发中，尚未上线，没有任何用户。已完成核心功能的技术选型和基础架构搭建，正在进行前端界面开发和后端 API 集成。

### MVP 版本核心目标
1. 实现食材、厨具、调料的智能匹配功能，准确率达到 80% 以上
2. 完成 AI 菜谱生成功能，能够根据用户输入生成符合要求的创意菜谱
3. 提供基础的菜谱管理和个人辅助功能，确保核心用户流程完整可用

## 项目背景与痛点分析

### 3.1 项目起源
作为一名普通上班族，我经常面临这样的场景：下班后打开冰箱，发现里面有一些零散的食材，比如半颗白菜、几个鸡蛋、一块豆腐，但不知道这些食材能做什么菜。打开常用的菜谱APP，要么是推荐各种需要特定食材的热门菜谱，要么是搜索结果与现有食材匹配度很低，最后只能随便煮个面条应付了事。

尝试了市面上主流的菜谱产品后，我发现它们普遍存在一个问题：都是以菜名为中心，用户需要先想到要做什么菜，然后查看需要哪些食材，而不是反过来根据现有食材推荐菜谱。这种模式在"不知道吃什么"的场景下完全不适用。

这个痛点不仅困扰着我，也困扰着许多忙碌的现代人。据统计，每个家庭每周平均会浪费约 5 公斤食材，其中很大一部分原因就是不知道如何利用现有食材。这不仅造成了经济损失，也对环境造成了负担。

### 3.2 现有解决方案的核心缺陷
| 产品类型 | 核心功能 | 用户体验 | 适用场景 | 主要缺陷 |
|---------|---------|---------|---------|---------|
| 传统菜谱APP | 按菜系、菜名浏览 | 内容丰富但匹配度低 | 明确知道要做什么菜 | 无法根据现有食材推荐 |
| 社区分享平台 | 用户分享菜谱 | 内容多样但质量参差不齐 | 寻找创意菜谱 | 缺乏智能匹配功能 |
| 外卖平台 | 直接点餐 | 方便快捷但成本高 | 不想做饭 | 无法解决食材利用问题 |

现有产品的核心缺陷在于：
1. **食材反向匹配能力弱**：用户无法快速找到使用现有食材的菜谱
2. **忽略厨具限制**：推荐的菜谱可能需要用户没有的厨具
3. **调料信息不清晰**：未明确标注所需调料，导致烹饪过程中发现缺少材料

### 3.3 市场机会与趋势
根据国家统计局数据，2025年中国家庭烹饪市场规模达到 1.2 万亿元，年增长率保持在 5% 以上。随着人们对健康饮食的重视和生活节奏的加快，如何高效利用现有食材、减少浪费成为越来越多人关注的问题。

同时，AI 技术的发展为菜谱工具带来了新的可能性。根据艾瑞咨询的报告，AI 在餐饮行业的应用正在从简单的推荐系统向更智能的个性化服务转变，预计到 2027 年，AI 菜谱生成市场规模将达到 100 亿元。

当前市场上缺乏一款纯粹、高效、以食材为中心的工具类产品，大多数产品仍然停留在传统的菜谱浏览模式，无法真正解决用户"有食材不知道做什么"的核心痛点。

## 产品核心解决方案

### 4.1 产品整体逻辑
用户打开"我要揍饭"后，首先输入家中现有的食材，然后选择自己拥有的厨具，最后勾选现有调料。系统会先在菜谱库中进行智能匹配，找到符合条件的菜谱。如果匹配结果不足，系统会利用 AI 生成个性化的创意菜谱。用户可以查看菜谱详情，包括食材清单、烹饪步骤、营养分析等，也可以收藏、分享喜欢的菜谱。系统会记录用户的烹饪历史和偏好，为后续推荐提供参考。

### 4.2 核心功能详解

#### 智能菜谱匹配
- **解决的问题**：用户不知道如何利用现有食材
- **使用方式**：用户输入食材名称，系统自动识别并标准化处理
- **具体价值**：快速找到符合现有食材、厨具和调料的菜谱，减少食材浪费

#### AI 菜谱生成
- **解决的问题**：现有菜谱库中没有匹配项
- **使用方式**：系统根据用户输入的食材、厨具和调料，生成个性化的创意菜谱
- **具体价值**：提供更多选择，激发用户烹饪创意

#### 菜谱管理
- **解决的问题**：难以管理和查找收藏的菜谱
- **使用方式**：用户可以浏览、搜索、收藏、分享菜谱
- **具体价值**：方便用户管理个人菜谱库，快速找到常用菜谱

#### 个人辅助功能
- **解决的问题**：用户偏好和历史记录无法得到有效利用
- **使用方式**：系统记录用户的饮食偏好、烹饪历史，自动生成购物清单
- **具体价值**：提供个性化推荐，简化用户购物流程

#### 基础体验优化
- **解决的问题**：不同设备上的体验不一致
- **使用方式**：产品适配不同设备屏幕，提供实时交互反馈
- **具体价值**：确保用户在任何设备上都能获得良好的使用体验

### 4.3 MVP 版本功能边界

#### 一定会包含的功能
- 食材输入和智能匹配
- 厨具选择和适配
- 调料勾选和提示
- 菜谱库浏览和详情查看
- 基础的 AI 菜谱生成
- 菜谱收藏和分享
- 简单的用户偏好设置
- 多设备适配

#### 暂时不会包含的功能
- 复杂的用户账户系统（仅本地存储用户数据）

- 智能购物清单导出

原因：MVP 版本优先保证核心功能的稳定性和用户体验，复杂功能将在后续版本中逐步实现。

## 产品特色与差异化优势

### 第一个核心优势：食材优先的匹配逻辑
与现有"菜名优先"的产品不同，"我要揍饭"以食材为中心，用户只需输入现有食材，系统会自动匹配适合的菜谱。这种反向匹配模式更符合用户"有食材不知道做什么"的真实场景，能够显著减少食材浪费。

### 第二个核心优势：智能厨具适配
系统会考虑用户实际拥有的厨具，智能适配菜谱要求。例如，如果用户没有烤箱但有空气炸锅，系统会推荐可以用空气炸锅替代烤箱的菜谱，或调整烹饪步骤以适应现有厨具，确保用户能够实际执行菜谱。

### 第三个核心优势：纯粹高效的用户体验
产品专注于解决核心痛点，界面简洁直观，没有冗余功能和广告干扰。用户可以在30秒内完成食材输入，10秒内获得匹配结果，整个过程流畅高效，让用户能够快速找到解决方案。

## 技术实现方案

### 6.1 整体技术架构
项目采用前后端分离的技术架构，前端负责用户界面和交互，后端负责数据处理和 AI 调用。前端通过 API 与后端通信，后端与第三方 AI 服务交互获取生成内容。数据存储采用本地存储和轻量级数据库相结合的方式，确保用户数据的安全性和可用性。

### 6.2 核心技术选型
- **前端**：React + TypeScript + Tailwind CSS
  - 理由：React 生态成熟，TypeScript 提供类型安全，Tailwind CSS 简化样式开发
- **后端**：Node.js + Express
  - 理由：轻量级、易于部署，适合个人开发
- **数据库**：SQLite
  - 理由：文件型数据库，无需额外配置，适合个人项目
- **AI 服务**：调用第三方大模型 API
  - 理由：快速实现 AI 功能，无需投入大量资源开发自研模型

### 6.3 AI 能力实现
AI 能力通过调用第三方大模型 API 实现，主要用于菜谱生成。为保证生成内容的质量和一致性，系统会：
1. 构建标准化的 prompt 模板，确保 AI 理解用户需求
2. 对生成的菜谱进行结构化处理，提取食材、步骤、营养等信息
3. 进行基本的质量检查，过滤明显不合理的内容

### 6.4 性能与体验优化
- **前端优化**：采用代码分割、懒加载、缓存策略，减少首屏加载时间
- **后端优化**：使用缓存减少重复计算，优化 API 响应速度
- **交互优化**：添加实时加载状态、过渡动画，提升用户体验
- **适配优化**：采用响应式设计，确保在不同设备上的良好体验

### 6.5 数据安全与隐私保护
- **数据存储**：用户数据优先存储在本地，敏感信息不上传
- **AI 调用**：在调用 AI API 时，只传递必要的食材和厨具信息，不传递个人身份信息
- **隐私政策**：明确告知用户数据使用方式，遵守相关数据安全法律法规
- **数据加密**：对存储的用户数据进行加密处理，防止未经授权的访问

## 风险分析与应对措施

### 单人开发进度风险
- **风险点**：单人开发工作量大，可能导致进度延迟
- **应对措施**：采用敏捷开发方法，优先实现核心功能；使用 AI 辅助编程提高开发效率；合理规划开发时间，避免过度承诺

### AI 生成内容准确性风险
- **风险点**：AI 生成的菜谱可能存在食材搭配不合理、步骤不清晰等问题
- **应对措施**：建立质量检查机制，对生成的菜谱进行基本验证；收集用户反馈，持续优化 prompt 模板；提供用户修改和反馈功能

### 用户增长缓慢风险
- **风险点**：作为个人项目，缺乏推广资源，用户增长可能缓慢
- **应对措施**：通过社交媒体、美食论坛等渠道进行低成本推广；优化 SEO，提高搜索引擎排名；鼓励用户分享，利用口碑效应

### API 成本风险
- **风险点**：调用 AI API 需要付费，随着用户量增加，成本可能上升
- **应对措施**：设置合理的 API 调用频率限制；优化 prompt 设计，减少不必要的调用；考虑使用开源模型降低成本

### 数据安全风险
- **风险点**：用户数据可能面临泄露或滥用的风险
- **应对措施**：加强数据加密和访问控制；遵守相关数据安全法律法规；定期进行安全审计和更新
