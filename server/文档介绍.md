# server

## 1. 这个文件夹是干嘛的？（大白话）
这是一个后端服务器文件夹，使用 Node.js 开发，主要用来处理前端的请求，比如数据存储、API 接口、业务逻辑等。简单来说，就是前端页面上的功能需要后端来支持，比如用户登录、数据查询、保存数据等，这些都是由这个文件夹里的代码来处理的。

## 2. 里面存放哪些内容？
- **config/**：配置文件，比如数据库配置等
- **data/**：数据文件，包括数据库文件、JSON 数据等
- **docs/**：文档，比如 API 文档
- **lib/**：库文件，比如数据库连接等
- **logs/**：日志文件，记录服务器运行情况
- **middleware/**：中间件，处理请求的中间环节
- **public/**：静态文件，比如调试页面等
- **routes/**：路由文件，处理不同的 API 请求
- **scripts/**：脚本文件，比如数据处理、生成数据等
- **services/**：服务文件，比如 AI 服务、计算服务等
- **utils/**：工具函数
- **配置文件**：package.json（项目依赖）、.env（环境变量）等

## 3. 哪些文件绝对不能删？
- package.json：记录项目依赖，删了就不能安装包了
- index.js：服务器入口文件，删了服务器就无法启动
- config/：配置文件，删了服务器可能无法正常运行
- data/：数据文件，删了数据就没了
- routes/：路由文件，删了 API 接口就用不了
- lib/：库文件，删了服务器可能无法正常工作
- .env：环境变量文件，删了可能导致配置丢失

## 4. 哪些是备份/旧版本/无用文件？
- index.js.backup：备份文件，是 index.js 的备份
- 各种脚本文件（scripts/ 目录下）：有些可能是临时使用的，比如数据迁移、测试等
- logs/ 目录下的日志文件：可以定期清理
- data/ 目录下的备份文件，比如 菜谱备份_422道.zip

## 5. 这个文件夹是否影响项目正常运行？
是的，非常影响。这是后端服务器的代码，如果这个文件夹有问题，前端的功能就无法正常工作，比如无法登录、无法获取数据、无法保存数据等。

## 6. 我可以在这里做什么？
- 修改 API 接口逻辑
- 添加新的 API 接口
- 调整数据库配置
- 添加新的服务功能
- 运行脚本处理数据
- 查看日志排查问题

## 7. 注意事项（非常重要）
- 不要删除核心配置文件和数据文件，否则项目可能无法运行
- 修改代码后，需要重启服务器才能生效
- 如果添加了新的依赖包，需要运行 npm install 安装
- 数据库文件（.db 文件）不要随意删除，否则数据会丢失
- 环境变量文件（.env）不要泄露，里面可能包含敏感信息
- 运行脚本时要注意，有些脚本可能会修改数据
